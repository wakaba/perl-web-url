NAME

Web::DomainName::Canonicalize - Domain name canonicalization

SYNOPSIS

  use Web::DomainName::Canonicalize;
  $out = canonicalize_domain_name $in;
  $out = canonicalize_url_host $in;

DESCRIPTION

The Web::DomainName::Canonicalize module provides functions to convert domain names into their canonical representations, as Web browsers do.

FUNCTIONS

This module exports two functions:

$out = canonicalize_domain_name $in

Canonicalize a domain name, i.e. dotted labels. The input might contain non-ASCII characters; they would be Punycode-encoded by the function. If the input has no canonical representation (i.e. the input contains a character that cannot be used in domain names) the function returns undef. Note that the function might still return a non-undef value when the domain name is invalid but it has a canonical representation.

Input is considered as a domain name; no IP address canonicalization is performed.

$out = canonicalize_url_host $in

Canonicalize the host component of a URL. In addition to canonicalize_domain_name, percent-encoding and -decoding, IP address canonicalization, and detection of some additional invalid characters are performed. If the input has no canonical representation (such that the URL is invalid), the function returns undef.
